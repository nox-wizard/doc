Detailed Description

Player interface

Players can use ingame commands, that is, they can send commands to the server to make various things. Not all commands are available to all players, every command needs a certain authorization to be used, this authorization is the "command level" or "privlevel".
The privlevel is a number, this number defines if a command is usable by a player or not.
To use a command the player has to write:
'[commandname]
for example:
'wipe 'kill 'invul
are all commands.

Usually a command takes some parameters wich specify how the command should work:
'invul on
'kill target
are examples.

Using commands

Using commands is easy, simpy type the command (with his parameters) and the follow the instructions that appear as messages in the bottom left corner of the screen.
Usually a command requires to target a character or item, in that case the target pointer will appear with a prompt saying what you should target. When a command requires parameters, there always some default values for the most used values.
When a parameter has a default value tou can ommit it in the command call, and the default will be assumed.
If you want to specify a parameter, you have to specify all the preceding parameters, or you will get errors.
In the commands documentation, it is always written the dafault value for every parameter.

Command areas

A player who can use the 'area command, can set "command area".
This is needed sometimes when you have to aplly a command to a large number of items/characters and it would be boring to write the command many times.
See the 'area for more details.

Customizability

You can define all commands you want, modify the privlevels, modify/erase existing commands, in other words, you can do everything you want with commands!
You can add a command by adding an entry in small-scripts/commands.txt.
A command entry in commands.txt follows this syntax:
command,priv,function
  • command is the command name
  • priv is the privlevel c character must have in order to use the command
  • function is the Small public function to be called when command is triggered

Then you will have to write a public function with the following proptotype:

        public function(const chr)

in one of your included files.
When a character says 'commandname ('add 'kill ...) your function will be called.

How the system works

Every time a player says something, the speech is checked by the sources and if it begins with the triggering character (' by default) and if the character has the right privlevel, the Small function is called.
The speech is automatically subdicided into prameters, every word except the first (the command name) is taken as a parameter and stored in CP_STR_PRIVLEVEL,there are 8 parameters available.
In the callback function parameters can be read with readCommandParams(chr) wich reads CP_STR_PARAM and stores the parameters in the __cmdParams[][] buffer wich can be used in scripts to access parameters.
Note that in __cmdParams[][] all parameters are strings, so if your command takes numerical parameters you will have to convert strings into integers.
Anyway, the better way to learn scripting commands is to browse existing ones and take example, some commands scrpits are very complicated, try and understand simple ones first.
