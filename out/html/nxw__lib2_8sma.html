chr_canMoveItem (  const  chr,
const  itm,
const  msg
) 
 

Author:
Fax
Parameters:
chr: the character
itm: the item
msg: true to activate messages
Since:
0.82
If msg == true then the character will receive messages in case the check fails, explaining why he can't move the item.
If the item is too far the function returns -1, if it can't be moved by the character the function returns 0, if it can be moved the function returns a number > 0.
In case of success the returned value is the ratio between the maximum weight the character can handle and the item's weight, in tenths (10 = 1, 15 = 1.5, 34 = 3.4 and so on).
See cmd_flip(const chr) for an example of how is this used.

Returns:
-1, 0 or weight ratio

file_scan (  filename  [],
callback  [],
exclude  []
) 
 

Author:
Fax
Parameters:
file[]: the file to be scanned, path starts from NOX folder.
callback[]: the function to be called for each line
exclude[]: list of "comment" charatcers
Since:
0.82
The file is scanned line by line and calls the callback for each line.
If the line starts with one of the tokens specified in the exclude string, the line is skipped, this allows toput comments in files.

The callback will be passed the file handle and the line number, read the line with file_getScanLine().
line number = INVALID means EOF (End Of File). A typical callback will be structured as follows:

public process_line(file,line)
{
        if(line == INVALID)
        { 
                printf("EOF^n");
                return;
        }
        
        printf("scanning line %d^n",line);
        
        new line[FILE_SCAN_BUFFER_SIZE];
        file_getScanLine(line);
}

file_scan(file,"process_line","//")

Returns:
the number of scanned lines or INVALID if file does not exist

getRectangle (  const  chr,
callback  []
) 
 

Author:
Fax
Parameters:
chr: the character
callback[]: the function that will be called at the end of rectangle aquisition
Since:
0.82
This function is used to ask a player for a map rectangle, player hasto target 2 map locations (he can target items or characters, in that cas the object's position will be taken) and then the callback will be called and it will be passed the rectangle.
Callback proptotype must be:
public callback(chr,x0,y0,x1,y1)
x0,y0 are the top left corner coords
x1,y1 are the bottom right corner coords
It's always x0 <= x1 && y0 <= y1.
Returns:
nothing
