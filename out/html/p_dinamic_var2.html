Come avere infinite variaibli a disposizione

Overrides, callbacks and event handlers allow us to make the emulator execute our custom scripts, without them all the small code we write would never be executed.

CALLBACKS A callback is a public small function that is called by the emulator. Since the function is called by the source code, you don't have control on the parameters passed to it, so the prototype of a callback is determined and depends on the use of the callback. Every NOX feature that requires callbacks will provide the prototype to be used.

OVERRIDES overrides are declared in override.scp and are used to execute custom code wich will extend or bypass the standard source code. If you open override.scp you will see there are some sections, identified by [name], every section is about a particular aspect of the source code. The only sections you have to worry about are:

  • [skilltargets]
  • [skills]
  • [special]
  • [speech]
other sections are either unused or there is a bettere way to call your script (with event handlers for example)
