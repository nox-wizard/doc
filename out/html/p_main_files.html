Main files

The override.* files

The "scripts\small-scripts" folder contains 3 important files:
  • override.sma
  • override.scp
  • override.amx

These files are the MOST important files for NOX, you can mess up with other files, but NEVER rename/delete these files (of course you can try, your PC won't explode, but NOX won't start).

override.sma

override.sma is the main Small file, NOX only looks into this file when he searches for Small scripts.
If you open it you will notice that it only contains #include directives ... uh! I forgot to say: I'm assuming you already know Small syntax so you know what an #include is, if you don't, close this page and study Small before continuing.

Welocome back, you now know everything about Small syntax right? ok.
We were saying that override.sma contains only #includes, there is a reason for this: we want to have scripts oredered in files,every script has is own file and we don't want to mix up things that do not have nothing in common.
override.sma is used to put all script files together and this is don by including them, so when the compiler parses override.sma it will automatically see all our scripts.

NEVER PUT CODE INTO OVERRIDE.SMA<br>
Now focus on: 
        #include "small-scripts/myshard.sma"

This includes the myshard.sma file wich is in the same folder as override.sma, your shard's scripts shoud be included in that file and not in override.sma, the reason is simple: when we update NOX scripts sometimes we modify override.sma, and if you include your scripts there and then update NOX with new scripts you may lost you script's link,this is not that big problem, you can always restore the lost #includes, but if you use myshard.sma you are sure you will never have trouble because that file is never modified.
So if you created a custom folder for you shards' scripts (and you should have done it),open myshard.sma and include your new folder.
The final rule is:

NEVER TOUCH OVERRIDE.SMA<br>
So why did I explain what override.sma is if you must never touch it? It was only to avoid you yelling in the forum: "Hey! I removed override.sma and now nothing works! it's a bug!"

override.scp

In the scripts-scripts there is another file called override.scp.
This is one of the most important files, and I'm not going to explain it in detail here, there will be another section for this.
For now just know that override.scp is used to tell the emulator when to call a Small script.
The fact that I only wrote a few lines on this file doesn't mean it's not important as override.sma, override.scp will be you nightmare for the next few weeks of study!

override.amx

This file is the ONLY important file for NOX, but it's the LESS importatnt file for you, it contains AMX code, you can open it and try to read it, but it's only some kBytes of ASCII nonsense.
Every time NOX compiles the scripts it creates a new override.amx and overwrites the old one, this file is then loaded and it's code executed.
NOX can run with only this file in the scripts-scripts folder, without caring of any other .sma file, this means that you can upload only this file on the server and it will run.
WAIT! Befaore trying it you must tell NOX that you want to use the current override.amx or it will seek for override.sma and will throw an error if it doesn't find it.
Open "config\server.cfg" and search: 
        USEINTERNALCOMPILER 1

and turn it to 0: this deactivates the Small compiler and makes NOX load override.amx without compiling the scripts.
This feature is useful if you don't want to upload all scripts to the remote server and also speeds up the server startup a lot because scripts compilation is skipped.
